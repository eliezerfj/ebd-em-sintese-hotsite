# Projeto: EBD em Síntese

Este projeto é a base da landing page **EBD em Síntese**, desenvolvida em HTML, Bootstrap 5 e JavaScript.  
A partir deste ponto, o objetivo é ajustar o visual e deixar o layout o mais fiel possível ao design original no Figma.

---

## Estrutura geral

O modelo está dividido em seções:

**Header**, **Hero**, **Countdown**, **Vídeo**, **Sínteses**, **Preletores**, **Aviso** e **Footer**.

Recomenda-se revisar como cada parte está organizada para manter essa mesma lógica em todo o projeto.  
Cada bloco foi planejado para permitir trabalhar separadamente e de forma organizada.

---

## Figma

Use o **Figma como referência principal** para ajustar fontes, tamanhos, cores, margens e espaçamentos.  
Você tem total **liberdade para modificar ou reorganizar o que for necessário**, desde que o resultado final seja o mais fiel possível ao design.  

Se perceber que algo pode ficar melhor visualmente, pode ajustar — o importante é que o resultado final pareça profissional e coerente com o Figma.

---

## Organização do projeto

Estrutura recomendada de pastas:
/ebd-em-sintese/
│
├── index.html
│
├── /assets/
│ ├── /css/ → style.css e responsive.css
│ ├── /js/ → main.js
│ ├── /img/ → logos, fundos, preletores etc.
│ └── /vendor/ → Bootstrap e Bootstrap Icons

**Instruções:**

- Transfira o conteúdo do `<style>` para o arquivo `style.css`.  
- O JavaScript (contador e outras funções) deve ficar em `main.js`.  
- As imagens e logos devem ser salvas em `/assets/img/`.  

Eu vou enviando os **logos e materiais visuais** conforme forem necessários.  
Enquanto isso, use **placeholders** ou imagens genéricas.  
Se você conseguir localizar os logos originais, ótimo — ainda não os tenho agora.

---

## Responsividade

Garanta que o layout esteja correto em **celulares, tablets e desktops**.  
Se precisar ajustar breakpoints, paddings ou tamanhos de texto para encaixar melhor, fique à vontade.  
A prioridade é que o site fique **limpo, equilibrado e fiel ao design** em qualquer dispositivo.

---

## Antes de enviar uma nova versão

Antes de me enviar a próxima atualização:

- Verifique se todos os **botões e links** estão funcionando.  
- Revise **fontes, cores e margens** de acordo com o Figma.  
- Confirme que o **menu mobile** e as seções se comportam corretamente ao rolar a página.
